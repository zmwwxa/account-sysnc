# 常见问题解答 (FAQ)

## 安全性问题

### Q: 这个工具有封号风险吗？

**A: 没有封号风险，完全安全。**

这个工具**只进行文件系统操作**，完全不涉及游戏进程或游戏数据修改。

### Q: 工具会读取游戏进程吗？

**A: 不会。**

工具的工作原理非常简单和透明：

#### 工具做什么：
1. ✅ **读取配置文件**：扫描 `userdata` 目录下的角色配置文件
2. ✅ **复制文件**：将配置文件从一个文件夹复制到另一个文件夹
3. ✅ **备份文件**：备份配置文件到 `backups` 目录

#### 工具不做什么：
- ❌ 不读取游戏进程或内存
- ❌ 不注入游戏程序
- ❌ 不修改游戏数据或逻辑
- ❌ 不拦截网络通信
- ❌ 不修改游戏客户端文件
- ❌ 不访问游戏服务器

### Q: 操作的是什么文件？

**A: 只操作玩家自己的配置文件。**

工具复制的都是**玩家可以自由管理的配置文件**：

```
userdata/[账号]/[大区]/[服务器]/[角色]/
├── custom.dat              # 界面布局和设置
├── custom.dat.addon        # 插件配置
├── hotkey.data             # 快捷键设置
├── hotkey_newback*.txt     # 快捷键备份
├── userpreferences.jx3dat  # 用户偏好设置
├── userpreferences/        # 偏好设置子文件夹
├── JT.data                 # 宏数据
├── JT_MacroContent.data    # 宏内容
└── 其他配置文件
```

这些文件**玩家自己也可以手动复制**，不受游戏保护。

### Q: 这和手动复制有什么区别？

**A: 没有本质区别，只是自动化了。**

使用这个工具 = 你手动执行以下操作：

```
手动操作步骤：
1. 打开资源管理器
2. 找到源角色文件夹
3. Ctrl+C 复制所有文件
4. 找到目标角色文件夹
5. Ctrl+V 粘贴
```

**如果手动复制配置文件不会封号，那么使用这个工具也不会。**

### Q: 官方允许这样操作吗？

**A: 允许。**

剑网三官方允许玩家：
- ✅ 备份和还原配置文件
- ✅ 在不同角色间共享界面设置
- ✅ 使用游戏官方支持的插件和宏
- ✅ 自由管理 userdata 目录下的文件

这个工具只是帮助你更方便地完成这些**官方允许的操作**。

### Q: 代码是开源的吗？可以审查吗？

**A: 是的，完全开源。**

项目结构清晰，所有代码都可以查看和审查：

```
account-sysnc/
├── frontend/           # 前端界面代码（React + Electron）
│   ├── src/           # 可以看到所有界面和逻辑
│   └── electron/      # Electron 主进程代码
└── backend/           # 后端逻辑代码（Python + Flask）
    ├── app.py         # API 接口
    ├── role_scanner.py    # 角色扫描
    ├── role_copier.py     # 文件复制
    └── backup_manager.py  # 备份管理
```

您可以自行检查代码，确认工具只进行文件操作。

### Q: 工具是否会上传我的数据？

**A: 不会。**

- 工具**完全本地运行**，不联网
- 所有操作都在您的电脑上完成
- 不会上传任何数据到服务器
- 不会收集用户信息

您可以在断网状态下使用工具，功能完全正常。

## 使用建议

### 安全使用建议

1. **关闭游戏再使用**
   - 避免文件冲突
   - 不是因为封号风险，而是防止文件被游戏锁定

2. **先测试后使用**
   - 先在测试角色上尝试
   - 确保配置正常后再大规模使用

3. **保留备份**
   - 工具会自动备份
   - 建议保留重要角色的手动备份

4. **正常游戏**
   - 工具不影响游戏运行
   - 可以随时使用，不用担心

### 兼容性说明

1. **跨账号复制**
   - 不同账号间复制时，部分账号相关配置可能不兼容
   - 这是配置文件的限制，不是工具的问题

2. **插件数据**
   - 某些插件的数据可能依赖特定角色ID
   - 复制后可能需要重新配置部分插件

3. **服务器差异**
   - 不同服务器间的某些配置可能存在差异
   - 建议在同服务器角色间优先使用

## 技术原理

### 工具的实现方式

1. **前端（Electron + React）**
   - 提供用户界面
   - 发送 HTTP 请求到本地 Flask 服务

2. **后端（Python + Flask）**
   - 接收前端请求
   - 使用 Python 标准库的文件操作函数
   - 返回操作结果

3. **文件操作**
   ```python
   # 示例代码（简化版）
   import shutil
   from pathlib import Path

   # 复制文件
   shutil.copy2(source_file, target_file)

   # 复制整个目录
   shutil.copytree(source_dir, target_dir)
   ```

### 不涉及的技术

- ❌ 进程注入（Process Injection）
- ❌ 内存读写（Memory Manipulation）
- ❌ API Hook
- ❌ DLL 注入
- ❌ 网络拦截
- ❌ 游戏客户端修改

**工具只使用最基础的文件操作，和你用资源管理器复制文件一样。**
